## 2. Implementation

### 2.1 Stochastic Policy Gradient and TD Error in A2C

Task 1 的 A2C 使用 actor-critic 架構。Actor 負責輸出 stochastic policy，critic 負責估計狀態價值函數。

在連續動作空間中，policy 不能直接輸出 discrete probability，因此 actor 輸出 Gaussian distribution：

```text
pi(a|s) = Normal(mu_theta(s), sigma_theta)
```

在 `a2c_pendulum.py` 中，actor network 由兩層 hidden layer 組成，最後輸出 action mean。Pendulum 的 action 範圍是 `[-2, 2]`，因此 mean 經過 `tanh` 後乘上 2。standard deviation 由 learnable `log_std` 控制，並使用上下界限制，避免 exploration 過大或完全消失。

訓練時，agent 從 policy sample action：

```text
a_t ~ pi_theta(a_t | s_t)
```

並記錄：

```text
log pi_theta(a_t | s_t)
entropy(pi_theta(. | s_t))
```

critic 估計目前 state value：

```text
V_phi(s_t)
```

TD target 使用 n-step return：

```text
R_t = r_t + gamma r_{t+1} + ... + gamma^n V_phi(s_{t+n})
```

TD error 或 advantage 為：

```text
A_t = R_t - V_phi(s_t)
```

Actor loss 使用 policy gradient：

```text
L_actor = - E[log pi_theta(a_t | s_t) * A_t] - beta * H(pi_theta)
```

其中 entropy term 用來保留 exploration。Critic loss 使用 mean squared error：

```text
L_critic = E[(R_t - V_phi(s_t))^2]
```

這個設計使 actor 會增加高 advantage action 的機率，降低低 advantage action 的機率；critic 則學習更準確的 value baseline，降低 policy gradient variance。

### 2.2 PPO Clipped Objective

Task 2 與 Task 3 使用 PPO。PPO 與 vanilla policy gradient 最大的不同是：PPO 不直接讓 policy 大幅更新，而是限制新舊 policy 的機率比值，避免一次 update 破壞已學到的策略。

PPO 先在 rollout collection 階段記錄 old policy 的 log probability：

```text
old_log_prob = log pi_theta_old(a_t | s_t)
```

更新時重新用目前 policy 計算：

```text
new_log_prob = log pi_theta(a_t | s_t)
ratio = exp(new_log_prob - old_log_prob)
```

PPO clipped surrogate objective 為：

```text
L_clip = E[min(
  ratio * A_t,
  clip(ratio, 1 - epsilon, 1 + epsilon) * A_t
)]
```

在程式中 actor loss 實作為：

```text
actor_loss = -mean(min(surr1, surr2)) - entropy_weight * entropy
```

其中：

```text
surr1 = ratio * advantage
surr2 = clipped_ratio * advantage
```

這個 clipping 設計可以避免 policy ratio 過度偏離 1。當 update 過大時，clipped term 會限制 objective 的增益，使 PPO 比 A2C 更穩定。

### 2.3 Generalized Advantage Estimation

PPO 使用 GAE 估計 advantage。TD residual 定義為：

```text
delta_t = r_t + gamma V(s_{t+1}) - V(s_t)
```

GAE 將多個時間步的 TD residual 做 exponentially weighted sum：

```text
A_t^GAE = delta_t + gamma lambda delta_{t+1}
          + (gamma lambda)^2 delta_{t+2} + ...
```

其中 `lambda` 對 bias-variance tradeoff 很重要：

- `lambda` 較小：variance 較低，但 bias 較高。
- `lambda` 較大：bias 較低，但 variance 較高。

本實驗在 Pendulum 與 Walker2d 都使用 `tau = 0.95`。實作上從 rollout 的最後一步反向計算 GAE：

```text
gae = delta + gamma * tau * mask * gae
return = gae + value
```

計算完 advantages 後再做 normalization：

```text
advantage = (advantage - mean) / (std + 1e-8)
```

這能讓 PPO update 的 scale 更穩定。

### 2.4 Sample Collection from Environment

三個任務的資料收集流程皆為 on-policy：

1. 以目前 policy 和 environment 互動。
2. 儲存 state、action、reward、done、value、log probability。
3. 累積到固定 rollout 長度或 episode 結束。
4. 使用這批 rollout 更新 actor 與 critic。
5. 更新後丟棄舊資料，重新用新 policy 收集資料。

A2C 使用較短的 n-step buffer，因此 update 較頻繁但 variance 較高。PPO 使用固定 rollout，例如 Task 3 最終設定使用 `rollout-len=2048`，再將 rollout 分成 mini-batch 做多個 epoch 的 update。

Task 3 另外實作固定 environment step snapshots：

```text
1M, 1.5M, 2M, 2.5M, 3M
```

這些 snapshots 用來呈現 Walker2d 在不同訓練步數下的 policy performance。程式會在 total environment step 超過指定 snapshot step 時自動儲存對應檔案。

### 2.5 Exploration in On-Policy Methods

雖然 A2C 和 PPO 都是 on-policy 方法，仍然需要探索才能找到好的 continuous-control policy。本實驗使用以下方式維持 exploration：

- Gaussian stochastic policy：訓練時 action 由 Normal distribution sample，不直接使用 mean action。
- Learnable `log_std`：policy 可以自行調整 action noise。
- `min_log_std` / `max_log_std` clamp：避免 noise 太小導致過早收斂，也避免 noise 太大導致學不到穩定步態。
- Entropy bonus：actor loss 加入 entropy term，鼓勵 policy 保持足夠隨機性。
- Evaluation 使用 deterministic mean action：訓練保留 exploration，測試時使用 mean action 取得較穩定表現。

Task 3 的實驗顯示，exploration 不能只是一味增加。entropy 太快降低會讓 agent 太早卡在次佳策略；entropy 太高或 policy update 太激進則會造成 critic loss 和 clip fraction 上升。最後採用中等 entropy coefficient 與較保守的 PPO clip 設定。

### 2.6 W&B Tracking

訓練過程使用 W&B 記錄。主要紀錄項目如下：

- `return`：每個 episode 的 training return。
- `best_return`：目前 training return 的最佳值。
- `actor loss`：policy update 的 loss。
- `critic loss`：value function 的 MSE loss。
- `entropy`：policy stochasticity，反映 exploration 程度。
- `approx_kl`：新舊 policy 的近似 KL divergence。
- `clip_fraction`：PPO update 中 ratio 被 clipping 的比例。
- `action/log_std`、`action/log_std_min`、`action/log_std_max`：policy noise 的變化。
- `eval/mean_reward`：periodic deterministic evaluation 的平均 reward。
- `eval/best_mean_reward`：目前 evaluation mean reward 的最佳值。

這些曲線讓我們能觀察 policy 是否穩定改善、exploration 是否過快消失、critic 是否失控，以及 PPO update 是否過大。

PPO 的 sample efficiency 在 Pendulum 上明顯優於 A2C。雖然 PPO 每次 update 的計算量較高，但同一批 rollout 可以經過多個 mini-batch epoch 重複使用，且 clipping 使 policy update 更穩定。因此它在較少 environment interaction 下達標。

A2C 的優點是實作簡單、update 直接，但缺點是 variance 較高，training return 容易震盪。PPO 的 clipped objective 和 GAE 則讓 update 比較平滑，尤其在 Walker2d 這種高維連續控制任務上差異更明顯。

### 3.5 Empirical Study: Clipping Parameter

PPO clip epsilon 控制 policy update 的最大幅度。epsilon 較大時 policy 可以更新更快，但也比較容易破壞目前策略；epsilon 較小時 update 較保守，通常更穩定。

在 Task 3 中，早期較大的更新容易造成 Walker2d 步態崩壞。後續設定逐步使用較保守的 clipping：

| Run | Epsilon | Observation |
| --- | ---: | --- |
| Early baseline | 0.2 | Update 較大，training return 有改善但 evaluation 不穩，1M 未通過。 |
| Stable runs | 0.1 - 0.11 | Update 較保守，搭配 GAE 與 entropy 調整後 evaluation 明顯穩定。 |
| Final run 10 | 0.11 | 1M 通過，best reward 超過 4000。 |

從曲線上看，合適的 epsilon 應該讓 `approx_kl` 保持低且穩定，`clip_fraction` 不應長期過高。若 clip fraction 持續升高，代表大量 samples 的 ratio 被 clipping，policy update 可能太激進或 batch/rollout 設定不夠穩。

### 3.6 Empirical Study: Entropy Coefficient

Entropy coefficient 決定 exploration 的強度。若 entropy 太低，policy 可能太早收斂，卡在只會站立或短距離移動的策略；若 entropy 太高，policy 可能長期太隨機，導致 critic target 變化大、evaluation 表現不穩。

Task 3 的主要比較：

| Run | Entropy Weight | Key Result |
| --- | ---: | --- |
| Run 8 frequent updates | 3.0e-3 | 1.5M 達到 3400.151，後期穩定，但 1M 未達標。 |
| Run 9 earlier discovery | 3.5e-3 | 1.5M 達到 2517.718，但 1M 仍未達標，後期 best 較低。 |
| Run 10 balanced early | 3.2e-3 | 1M 達到 2603.904，best 達到 4079.485。 |

Run 10 成功的原因是 entropy 衰退速度較平衡。它比 run 8 更鼓勵早期探索，但比 run 9 更不容易讓後期策略維持過高隨機性。因此 1M 可以通過，同時 best checkpoint 也能繼續提升。

## 4. Bonus Training Strategies

本次除了基本 A2C/PPO，也加入以下 training strategies：

- Eval-best checkpoint：週期性以 deterministic evaluation 測試模型，儲存 evaluation mean reward 最高的 checkpoint。
- Fixed-step snapshot：Task 3 自動儲存 1M、1.5M、2M、2.5M、3M 的模型，方便比較不同訓練階段的 performance。
- Log standard deviation bounds：限制 continuous action policy 的 exploration noise。
- Advantage normalization：讓 actor update scale 更穩定。
- Gradient clipping：限制 actor 和 critic 的 gradient norm，避免 training spike。
- Reward scaling：Task 1 A2C 使用 reward scaling 改善 critic target scale。
- Target-based training：Pendulum 嘗試用 target evaluation mean 作為 stopping criterion。

這些策略不是改變 PPO/A2C 的核心理論，而是讓實作在實際 continuous-control 環境中更穩定。
